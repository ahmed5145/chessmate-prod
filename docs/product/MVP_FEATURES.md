# MVP Feature List

## Must Have
- Select imported games manually.
- Shortcut to select the most recent N games.
- Start one batch analysis job.
- Show progress and completion state.
- Return one combined report for the batch.
- Surface recurring weaknesses.
- Summarize openings, middlegame, endgame, and time usage trends.
- Highlight a few critical moments that explain the main issue.
- Show strengths so the report does not read as purely negative.
- Produce a short coaching plan with next actions.

## Should Have
- Friendly coach-style copy instead of raw metric dumps.
- Clear prioritization of the biggest issue first.
- A concise summary card that tells the user what to focus on this week.
- A saved analysis history for previously analyzed batches.
- A retry action for failed or partial batch runs.

## Could Have
- Email notification when long-running analysis finishes.
- Comparison between two batches.
- Suggested training exercises based on the detected weakness.
- Batch tags such as "blunder-heavy" or "opening drift".

## Won't Have In MVP
- Full training curriculum builder.
- Subscription packaging complexity.
- Deep social features.
- Engine analysis reimplementation.
- Multiple report formats for different user levels.

## Report Content Checklist
- Overall batch summary.
- Top recurring weakness.
- Top recurring strength.
- Phase breakdown.
- Critical moments across the batch.
- Action plan.
- Confidence note when data is thin.

## UX Checklist
- No blank-looking report states when data exists.
- No generic coaching language that could apply to anyone.
- No over-focus on engine scores without explanation.
- No requirement that the user understand chess engine terminology to benefit.
